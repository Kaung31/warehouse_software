2Unified architecture — one system, many surfaces

Your concern is right: keep one source of truth. We add new surfaces (a PWA install for handhelds, a kiosk view for the workshop bench), but they all hit the same Next.js API routes, same Prisma schema, same Clerk auth.

Read this: the four boxes on top are the same web app rendered in different ways. The handheld scanner is just the Android Chrome PWA installed full-screen — DataWedge feeds barcodes as keystrokes into a global focus trap, so "scan to add to pallet" and "type RO‑id in CS form" are literally the same input.

7Scanner hardware playbook

Your instinct ("everything in one system") is the right one. Here's how to make a phone, a Zebra TC handheld and a desk barcode scanner all talk to the same web app — without forking the codebase.

1. Phone PWA (camera scan)

Install Scooter Hub from Chrome's "Add to Home Screen". Web app uses Barcode Detection API where available, falls back to html5-qrcode. Already partially wired in your /scan page.

  • Add Serwist (next-pwa successor) for offline shell
  • Web Vibration + Web Audio for feedback (you have audio)
  • Background sync queue for spotty Wi-Fi

2. Handheld (Zebra TC / Honeywell CT)

Run Chrome on the Android handheld, install your PWA full-screen. Configure DataWedge profile for "Chrome" with output mode = keystroke. Trigger pull → barcode characters appear in whichever input is focused.

  • Same UI as the phone — no native app to maintain
  • Global keyboard listener focuses a hidden input if nothing is focused
  • Suffix scan with Enter so React forms auto-submit

3. USB desk scanner (warehouse PC)

Cheap Honeywell/Datalogic in HID keyboard wedge mode. Plug into the warehouse Mac/PC, open Scooter Hub in Chrome, scanner types into whatever input. No drivers, no setup.

  • Best for high-volume inbound at the dock
  • Uses the same hidden-input focus trick as the handheld
  • Supports printing labels via DPD integration you already have
The trick that keeps it "one system": a top-level React provider (<ScannerProvider>) that always keeps focus in a hidden input and parses RO-… / PLT-… / SKU-… prefixes, then dispatches to the right module. Camera, handheld and USB scanner all funnel through this single code path. No more "scanner app" vs "web app" split.
